Implementation Master Plan

The Journey to a
Smart Campus.

From zero to a fully distributed cloud-native architecture. Here is the step-by-step roadmap to building CampusOps.

Project Phases

8 Steps to Production

We build and verify each phase before moving to the next. After each phase, you have a working, testable increment — no more "everything at once" complexity.

1

Scaffolding & Infrastructure 🏗️

Goal: Get a running containerized API with a connected database. Zero business logic, just the skeleton.
  • ▸ Init Node.js + TS project (package.json, tsconfig.json)
  • ▸ Create Express app with /health check
  • ▸ Env config with Zod validation
  • ▸ Docker Compose (API + PostgreSQL + Redis)
  • ▸ Prisma setup + empty schema & Logging handler
✓ `docker compose up` → Responds on localhost:3000/health
2

Database & Models 🗄️

Goal: Full database schema from ERD, migrations, and seed data with demo accounts.
  • ▸ Define Prisma models for Branches, Users, Modules, Groups
  • ▸ Relational models: GroupStudent, Planning, Absence
  • ▸ Tracking models: Progress, Payment, Notification
  • ▸ Generate + run Prisma migrations
  • ▸ Seed data script with demo accounts (Admin, Scolarité, Prof, Student)
✓ Run `npm run db:seed` → 5 demo users synced
3

Authentication & Security 🔐

Goal: Secure JWT auth flow with RBAC. No one accesses anything without proper tokens + role.
  • ▸ JWT utilities & password hashing (bcrypt)
  • ▸ Auth service & controller (register, login, refresh)
  • ▸ JWT Verification Middleware
  • ▸ RBAC middleware (`requireRole()`)
  • ▸ Security hardening (helmet, cors, rate-limit, zod schemas)
✓ Register → Login → Get JWT → Access endpoints
4

Core CRUD APIs 📡

Goal: Full REST API for all business modules. Each module = controller + service + routes + validation.

4A: Entity Management

CRUD for Branches, Users, Modules, Groups, Group Enrollment.

4B: Business Logic

Logic for Planning, Absences, Progress %, Payments, Notifications.

✓ All endpoints working in Swagger UI at `/api/docs`
5

Frontend Dashboard 🎨

Goal: Professional React SPA connected to the backend API. Role-based views, real-time data, premium design.
  • ▸ Vite + React + TypeScript setup
  • ▸ Global CSS + Theme Design System
  • ▸ Auth Pages (Login/Register) & JWT storage
  • ▸ Dashboard layout, Calendar view, Admin tables
  • ▸ Visual progress bars, Charts, and Notification panel
✓ Fully working dashboard at localhost:5173
6

Integrations & Bots 🤖

Goal: Connect the backend to the outside world — Telegram, Email, OpenClaw.
  • ▸ Telegram bot setup + `/link` OTP flow
  • ▸ Bot commands: /today, /week, /absence
  • ▸ SMTP and IMAP Email integrations
  • ▸ OpenClaw webhook receivers
  • ▸ Scheduled Cron jobs for 7 AM planning notifications
✓ Bot responds & triggers work automatically
7+

Docs, Tests & Cloud 🚀

Goal: Production-ready quality, automated tests, and live deployment.
  • ▸ Swagger/OpenAPI annotations generated
  • ▸ Jest Unit & Integration test coverage
  • ▸ Deploy PostgreSQL to managed provider (Supabase/Neon)
  • ▸ Deploy API & Frontend to containers / Edge network
✓ Live URL mapped + Demo Video recorded
